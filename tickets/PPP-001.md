# Outcome

A clean checkout has authoritative product, implementation, security, design, rubric, and ticket documents plus a pinned Clojure/ClojureScript build skeleton. Shared schemas reject malformed provider results and protocol messages before subsystem work begins.

# Scope

- Add all source-of-truth documents and traceability rules.
- Pin JVM, Clojure, ClojureScript, npm, test, and formatting dependencies.
- Define `bb` tasks and the single `bb verify` contract.
- Define protocol version, session format version, capability version, provider result schema, WebSocket envelope, and manifest schema.
- Add literal static host HTML and base CSS without product behavior.
- Add fake fixture conventions used by later tickets.
