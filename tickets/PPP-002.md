# Outcome

An unauthenticated visitor can exchange a fragment-delivered access code and reach a literal blank white host with a small immutable handle. The host can bootstrap and create/select sessions without exposing technical setup.

# Scope

- Signed access cookie with expiry and constant-time verification.
- CSRF token issuance and validation.
- Origin policy for HTTP and WebSocket upgrade.
- Browser fragment exchange and immediate fragment removal.
- `/api/bootstrap`, `POST /api/sessions`, and session selector wiring.
- Literal blank initial frame and accessible immutable handle.
- Development-only safe defaults; production startup validation.

# Acceptance Criteria

- [x] Access code never enters query parameters, server logs, browser storage, or later URLs.
- [x] Valid exchange sets HttpOnly, SameSite=Strict, Secure-in-production cookie.
- [x] Invalid or missing code cannot bootstrap.
- [x] State-changing requests without valid CSRF fail.
- [x] After access, canvas pixels are white except for the handle when sidebar is closed.
- [x] Handle is a labeled keyboard-operable button.
- [x] `+` creates a persistent session and selector can choose it.
- [x] Production refuses missing/weak access or cookie secrets.

# Domain/PBT Tests

- Vary access codes and signatures; only exact constant-time-verifiable values authenticate.
- Vary methods, origins, and CSRF values across state-changing routes.
- Playwright checks roles, URL fragment removal, cookie flags through context APIs, and blank outcome.
- Do not assert handle pseudo-element geometry or exact access guidance copy.

# Evidence

- access unit/integration test transcript
- Playwright access and blank-frame screenshot
- captured logs proving redaction
- `curl` examples with secret values replaced

# Out of Scope

- Real user accounts, logout UI, reset password, organizations, or invitations.
- Session rename and delete.
- Generated sidebar evaluation.
