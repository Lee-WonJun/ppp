# Outcome

Each UUID session owns a durable source tree, SQLite placeholder, manifest, history, checkpoint namespace, journal namespace, and staging area. History is canonical and append-only; current is a verified materialized view.

# Scope

- Safe session path construction and per-session fair lock.
- Create, list, load, and metadata update operations.
- Complete-file source materialization and SHA-256 manifest.
- Append-only history event directories with prompt/assistant/source/validation artifacts.
- Source/session/instance quota accounting without symlink traversal.
- Direct current-manifest load and source hash verification.
- Transcript storage and summary field without application logging.

# Acceptance Criteria

- [x] Session IDs must parse as UUID before path resolution.
- [x] All writes remain below `data/workspaces/local/sessions/<uuid>`.
- [x] New session starts at runtime version zero with fixed entrypoints.
- [x] History sequence is unique and append-only under concurrent attempts.
- [x] Current source hashes match manifest on every load.
- [x] Loading does not evaluate history events.
- [x] Quota exhaustion rejects new AI staging and preserves history/checkpoints.
- [x] Session switching returns independent source and SQLite paths.

# Domain/PBT Tests

- PBT-05: at least 1,000 generated identifiers and path fragments cannot escape.
- Concurrent history appends produce unique increasing event sequences.
- Random complete-file writes produce a manifest equal to file hashes.
- Quota boundary tests cover just below, equal, and above each limit.
- Do not test directory listing order unless the contract requires it.

# Out of Scope
