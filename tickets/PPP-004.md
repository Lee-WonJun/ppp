# Outcome

The kernel can generate one schema-valid turn through a no-tool, read-only Codex CLI process or a deterministic fake provider. Provider work is bounded, queued, observable without prompt/source logs, and resumable through a stored thread ID.

# Scope

- Provider protocol and normalized result model.
- Static JSON Schema plus host Malli validation.
- `ProcessBuilder` argument vector for Codex 0.144 behavior.
- Cleared environment, empty workdir, stdin context, bounded JSONL/final output.
- OAuth readiness and actionable setup error.
- Thread capture/resume and reset hook.
- Global/per-session FIFO queue, capacity eight, timeout 120 seconds.
- Fake scenarios for reply, clarify, demo changes, restore, invalid source, timeout, and security refusal.

# Acceptance Criteria

- [x] No shell string is used to launch Codex.
- [x] Command disables every tool and feature named in `docs/SPEC.md`.
- [x] Repository and session directories are not process working directories or readable context files.
- [x] Prompt/source enter stdin only.
- [x] Invalid JSON, invalid schema, oversized output, nonzero exit, and timeout become stable failures with no product change.
- [x] `thread.started` ID persists; resume targets the exact ID.
- [x] Restore can force a new thread.
- [x] Fake provider drives all non-live tests without network or OAuth.
- [x] Queue is FIFO, one global job runs, and ninth queued job is rejected.

# Domain/PBT Tests

- Generate provider results around every optional/required boundary.
- Use a fake executable to capture exact argv, cwd, environment keys, stdin, stdout, stderr, exit, and timeout behavior.
- Concurrent fake jobs prove FIFO and maximum active count one.
- Do not invoke live Codex in `bb verify`.
