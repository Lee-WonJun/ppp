# Outcome

The immutable Reagent host can evaluate generated client and sidebar source in fresh browser SCI contexts, render them inside separate ShadowRoots, keep stable state, and recover through an immutable handle and Safe Mode.

# Scope

- Host-owned Canvas, Sidebar, and hidden staging ShadowRoots.
- Client capability wrappers and stable Reagent page-state.
- Deterministic shared/client/sidebar evaluation and registration.
- Runtime CSS insertion scoped to roots.
- Hidden representative render and paint/error capture.
- Active registry swap only on activation.
- Default and last-success bundled sidebar.
- Handle controls and `Ctrl+Alt+Shift+P` Safe Mode.
- Action transport wrapper and event-value/prevent-default helpers.

# Acceptance Criteria

- [x] Generated client has no arbitrary DOM, window, storage, cookie, fetch, WebSocket, eval, Function, worker, or Node access.
- [x] Canvas CSS cannot hide or intercept the immutable handle.
- [x] Client source can replace page and sidebar without browser refresh.
- [x] Hidden stage error does not modify active page, sidebar, CSS, or state.
- [x] Successful activation causes Reagent to render the new registry.
- [x] Compatible page-state survives component replacement.
- [x] Safe Mode bypasses generated sidebar rendering and exposes recovery controls.
- [x] Desktop, 390px mobile, keyboard, focus, live region, and reduced-motion states pass.

# Domain/PBT Tests

- CLJS tests cover registry lifecycle and stable state behavior.
- Escape fixtures try global objects and arbitrary interop.
- Playwright deliberately stages a throwing sidebar and verifies fallback plus shortcut.
- Tests select roles/labels and state outcomes, not CSS classes or DOM nesting.
