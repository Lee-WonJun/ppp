# Outcome

A clean Linux amd64 container runs the compiled JVM/browser application as non-root with only two persistent volumes. CI uses the fake provider, verifies the project, builds the image, scans secrets, and can publish to GHCR without embedding runtime credentials.

# Scope

- Multi-stage Dockerfile for CLJS release and JVM dependency/package preparation.
- Non-root runtime image, read-only root compatibility, tmpfs guidance.
- Compose app with `ppp-data` and `codex-home` volumes.
- Device-auth commands for Codex OAuth.
- Health/readiness probes and graceful shutdown.
- GitHub Actions verify/build workflow and amd64 GHCR metadata.
- Packaged smoke script and container inspection.
- Backup, image rollback, session-format compatibility documentation.
- Coolify configuration documentation only.

# Acceptance Criteria

- [x] Image builds from clean checkout for `linux/amd64`.
- [x] Container runs as non-root and supports read-only root plus `/tmp` tmpfs.
- [x] Only data and Codex home volumes are writable.
- [x] Image contains no `auth.json`, `.env`, access code, session data, or test artifacts.
- [x] Fake-provider container reaches `/healthz`, `/readyz`, bootstrap, session create, and packaged browser asset.
- [x] Compose device-auth commands persist login in `codex-home`.
- [x] CI never receives or invokes owner OAuth.
- [x] Deployment backup/rollback steps are documented and locally rehearsed where possible.
- [x] No external deployment occurs without explicit approval.

# Domain/PBT Tests

- Packaged smoke tests observable endpoints and a deterministic fake change.
- Inspect UID/GID, architecture, writable mounts, image history, and forbidden filenames.
- Secret scan covers tracked files and exported image filesystem.
- Do not use a development server or bind-mount source as package evidence.

# Evidence

- `Dockerfile`, `docker-compose.yml`, and `.github/workflows/verify.yml`
- `scripts/docker_smoke.clj`: amd64, UID 10001, read-only root, two volumes,
  health/readiness/access/session/asset, SQLite, deterministic change, persistent
  Codex-home sentinel, quiesced data archive, fresh-volume restore, and restored
  runtime/checkpoint pass
- `scripts/secret_scan.clj`: repository and forbidden credential file scan pass
- SQLite native extraction remains executable under `/var/lib/ppp/.native` while
  `/tmp` stays a `noexec` tmpfs
- direct `docker compose run --rm app` rehearsal wrote a mode-0600 non-secret
  sentinel to `CODEX_HOME`, read it from a second disposable container, and removed
  the test volumes; this proves the documented interactive device-auth command uses
  persistent storage without copying or inspecting owner OAuth
- `bb docker-smoke` stopped the packaged app, archived `ppp-data`, restored it into
  a fresh named volume, restarted the same image, and reproduced the session runtime
  version, generated CSS, and checkpoint list

# Out of Scope

- Actual GHCR push if credentials/approval are absent.
- Actual Coolify/VPS deployment.
- ARM64 image.
- Kubernetes or multi-node operations.
