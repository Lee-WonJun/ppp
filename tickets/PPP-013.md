# Outcome

Generated canvas and sidebar code runs in an opaque-origin sandbox frame rather than the authenticated parent window. Inside that frame it can use ordinary browser APIs and JavaScript interop for games, canvas, keyboard input, animation, audio, and other product behavior. The immutable parent keeps authentication, sessions, commit coordination, and recovery controls.

# Scope

- A separately compiled sandbox-frame host and a typed `postMessage` bridge.
- Hidden-frame source evaluation and render staging before the requesting-tab ACK.
- Browser API and interop access inside the sandbox, with no parent DOM or credential access.
- Bridged server actions, sidebar conversation events, runtime errors, and serializable page state.
- Atomic frame activation, state handoff, disposal, rollback, resync, and Safe Mode.
- Migration of existing ShadowRoot browser tests and documentation.

# Acceptance Criteria

- [x] Generated code can use timers, keyboard events, Canvas/WebGL-compatible DOM access, animation frames, and normal in-frame JS interop without adding one host capability per feature.
- [x] The sandbox has an opaque origin and cannot read the parent DOM, cookies, local storage, CSRF token, or host JavaScript objects.
- [x] A staged frame renders both canvas and sidebar before ACK; failure preserves the active frame and SQLite/source hashes.
- [x] Server actions and host sidebar controls work only through validated bridge messages.
- [x] Compatible serializable page state survives frame replacement; nonserializable local objects remain frame-local.
- [x] Runtime replacement, rejection, restore, resync, session switch, and Safe Mode dispose timers/listeners by destroying the old frame.
- [x] A generated Tetris implementation automatically falls, accepts keyboard input, and remains recoverable after a rejected change.
- [x] Browser errors identify loading, evaluation, initial render, bridge, or runtime callback failure without exposing secrets.

# Domain/PBT Tests

- Bridge envelopes reject wrong channel, source window, session, version, request, and payload shapes.
- Rejected frame staging leaves active version and serialized state unchanged.
- Activation transfers only bounded structured-clone-safe state.
- Repeated stage/reject/activate sequences leave exactly one active frame and no pending request.

# Evidence

- `e2e/runtime.spec.mjs`: 6.5-second cold frame load, timer-driven updates,
  keyboard, Canvas, WebAssembly, parent-origin isolation, replacement state,
  rejection, and Safe Mode. Every transition asserts a new opaque-frame
  channel, removal of the previous frame, and exactly one remaining frame.
- `e2e/coordinator.spec.mjs`: SQLite-backed gallery actions, rejection reason propagation, recovery, restore, and follower resync.
- `e2e/access.spec.mjs`: session switching destroys the previous product frame.
- `ppp.client.runtime-test/safe-mode-stage-does-not-evaluate-generated-sidebar`:
  Safe Mode does not evaluate the generated sidebar entrypoint.
- `test/ppp/shared/protocol_test.cljc` and `test/ppp/websocket_test.clj`: bounded rejection-detail contract and delivery.
- `e2e/coordinator.spec.mjs`: generated Tetris automatic fall, arrow-key movement,
  and continued operation after a rejected browser change.
- `live-eval/codex-tetris.spec.mjs`: real Codex OAuth source generation plus
  timer/keyboard source contract and time-varying rendered playfield.
- Focused verification: 120 JVM tests / 687 assertions, 17 CLJS tests / 73
  assertions, and eight Playwright tests pass. The last full `bb verify` also
  passed Docker package smoke and secret scan before this focused lifecycle
  change; the final gate will repeat the complete command.

# Out of Scope

- npm/JVM dependency installation at runtime.
- Native browser permissions granted without a user gesture.
- Public multi-tenant origin allocation and per-tenant network permission UI.
- Cross-origin isolation headers required by `SharedArrayBuffer`.
