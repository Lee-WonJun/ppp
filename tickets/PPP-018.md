# Outcome

The generated conversation composer accepts Korean and other composition-based
input without duplicating partial candidates. The isolated frame owns the
immediate draft while the user types, and the authenticated parent receives a
monotonic copy for submission and recovery without stale parent echoes
rewriting an active composition.

# Scope

- Give the fixed sandbox-frame shell immediate ownership of the displayed
  composer draft while preserving the generated sidebar props contract.
- Version draft updates across the frame bridge so a late parent echo cannot
  overwrite a newer local candidate.
- Keep parent-originated clears and recovery updates authoritative when their
  revision is newer.
- Make Enter submission read the frame-local draft while retaining the
  existing composition, busy, blank, Shift+Enter, and one-shot guards.
- Add a compiled Chromium regression using the browser's IME composition
  protocol and the owner's exact `간단한` reproduction.

# Acceptance Criteria

- [x] Incremental Korean candidates remain replacements of the active
  candidate and produce exactly `간단한`, never a duplicated prefix sequence.
- [x] A delayed parent sidebar-model echo cannot write an older draft into the
  active textarea.
- [x] IME confirmation does not submit; the next ordinary Enter submits the
  committed text exactly once through the existing turn route.
- [x] The textarea keeps focus throughout composition, and existing English,
  Shift+Enter, blank, busy, and Send-button behavior remains passing.
- [x] Three fresh browser contexts with fresh sessions pass the original
  semantic path; one delays the sandbox bundle beyond five seconds.
- [x] Paired host/frame bundles are promoted together to port 8787 only after
  focused and complete gates pass.
- [x] `bb verify` passes from an isolated clean candidate copy.

# Domain/PBT Tests

- Browser property: for every increasing IME candidate in the fixed Korean
  sequence, the visible value equals the current candidate rather than a
  concatenation of earlier candidates.
- Ordering property: parent draft revisions may confirm or advance frame state
  but can never roll it back.
- Submission property: composition confirmation produces zero turn requests;
  a subsequent ordinary Enter produces exactly one.

# Evidence

- Pre-fix Chromium reproduction on port 8787 transformed `간단한` into
  `가간간다간단간단하간단한` before commit.
- Instrumented trace captured React setting the live value from `가` back to
  stale `ㄱ` and then forward to `가` after a `host/sidebar-model` echo. The
  next candidate therefore began a new composition and appended `간`.
- The regression failed against the old stable bundles at the second candidate
  (`expected "가", received "ㄱ가"`) and then passed against the candidate
  bundles in three fresh contexts, including a 6.5-second delayed fresh frame.
- Isolated `bb verify` passed 132 JVM tests/750 assertions, 21 CLJS tests/89
  assertions, Chromium 17/17, release builds without warnings, Linux amd64
  Docker smoke, and a 158-file secret scan.
- Verified host and frame release hashes were promoted together: host
  `feb630f9511cda5258c2509174be216fbfef30fcb26e64bde0c616137702024b`,
  frame
  `847b824bbd3180dc7d6903a766ccd6a2fe685ecf71663adaa438a802b492b601`.
- Stable `localhost:8787` passed the same three fresh-context IME paths, one
  with the 6.5-second frame delay. Four validation sessions were then deleted;
  the restarted Codex-ready JVM reports zero sessions, runtimes, connections,
  and pending stages.

# Out of Scope

- Changing provider, coordinator, or generated source capabilities.
- Replacing the generated sidebar with immutable host UI.
- Exposing technical bridge or revision details in the product language.
- Migrating released session formats or incrementing `capability-version`.
- External push, deployment, image publication, or Devpost submission.
