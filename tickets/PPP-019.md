# Outcome

While a turn is running, the conversation surface shows one calm, bounded
status line that combines the real kernel phase with a short user-facing
description. A visually animated ellipsis makes ongoing work legible without
streaming private provider reasoning or turning the conversation into an event
log.

# Scope

- Derive one plain-language detail from each existing progress phase in the
  authenticated host.
- Pass the phase label and detail through the generated-sidebar model.
- Render a fixed-width, zero-to-three-dot animation in the default, fake-provider,
  and immutable fallback sidebars without repeatedly changing live-region text.
- Keep the status to one visual line with graceful truncation on narrow
  surfaces.
- Provide a stable accessible status label and a static reduced-motion state.
- Add focused compiled-browser coverage for the phase copy, animation contract,
  fallback surface, and reduced-motion behavior.

# Acceptance Criteria

- [x] `Generating`, `Validating`, `Applying`, and `Applied` each show a distinct
  detail that truthfully describes that kernel phase.
- [x] Ongoing work visibly cycles through zero, one, two, and three dots without
  changing application state or emitting repeated protocol progress messages.
- [x] The visible status remains a single line and truncates its detail rather
  than resizing the sidebar.
- [x] Assistive technology receives one stable status label per phase; animated
  dots are hidden from it.
- [x] Reduced-motion mode disables the cycling animation and displays a static
  ellipsis.
- [x] Default, fake-provider floating, and immutable fallback sidebars use the
  same status contract.
- [x] No provider reasoning, diagnostics, generated source, or internal
  implementation language reaches the browser status.
- [x] Focused browser tests and `bb verify` pass from an isolated candidate
  copy before paired browser bundles are promoted.
- [x] Breaking pre-release browser-contract promotion resets development
  sessions and restarts the stable port 8787 JVM with an empty runtime registry.

# Domain/PBT Tests

- Phase mapping property: every public protocol phase maps to exactly one
  non-empty bounded label/detail pair; unknown phases fall back to neutral
  working language without exposing payload details.
- Accessibility property: the status accessible name is stable while the
  purely visual dots animate.
- Reduced-motion property: the same semantic status remains visible with no
  cyclic animation.

# Out of Scope

- Streaming chain-of-thought, provider JSONL, hidden reasoning, or raw
  diagnostics.
- A multi-line activity transcript or exact internal sub-step timeline.
- Adding new WebSocket progress phases or changing provider output schemas.
- External push, deployment, image publication, or Devpost submission.
