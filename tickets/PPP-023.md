# Outcome

When a generated product interaction fails, the next AI turn can inspect a
small, current, non-secret evidence set from the exact active sandbox frame
through an optional Codex Skill. Normal turns remain free of diagnostic text,
and parent-window or extension noise never enters the evidence set.

# Scope

- Capture bounded action, runtime, Promise, console, and direct-network
  failures inside the active opaque-origin frame.
- Carry a volatile deduplicated ring from frame to authenticated host and only
  with the next turn to the coordinator/provider boundary.
- Install a temporary progressively disclosed provider Skill only when records
  exist; keep the records out of the ordinary prompt, source, transcript,
  history, and logs.
- Redact and validate at both browser and server boundaries.
- Prove the reported `auth/register` 400 path and parent extension-noise
  exclusion in a real compiled browser.

# Acceptance Criteria

- [x] A failed generated action records kind, action ID, stable code, status,
  and bounded user-safe message when available.
- [x] Active-frame `window.error`, unhandled rejection, `console.warn` /
  `console.error`, and direct `fetch` rejection/non-success status produce only
  allowlisted bounded fields.
- [x] The parent accepts diagnostics only from the exact active frame; staging,
  replaced, parent, and browser-extension events are ignored.
- [x] The host retains at most 12 deduplicated records and clears them on
  project or runtime activation.
- [x] `/turns` accepts only an optional strictly normalized diagnostic vector;
  malformed or oversized input fails before provider admission.
- [x] Diagnostics flow transiently through the coordinator and never enter
  history events, transcript summary, application logs, or persisted session
  files.
- [x] `ppp-client-diagnostics` exists in the isolated Codex job directory only
  when records exist; the records are absent from provider stdin and the Skill
  labels them untrusted evidence rather than instructions.
- [x] The packaged validation Skill points Codex to the optional diagnostic
  Skill only for a relevant broken-product request and grants no new tool or
  runtime authority.
- [x] A compiled-browser regression reproduces `auth/register` HTTP 400,
  captures `Use a valid sign-in identifier.`, and proves a parent-window error
  does not enter the active-product ring.
- [x] User-visible UI continues to use product language and never exposes
  console, network, stack, Skill, model, or diagnostic internals.
- [x] Focused JVM/CLJS/browser gates and `bb verify` pass before paired browser
  bundles are promoted to stable port `8787`.
- [x] A breaking pre-release browser/runtime contract change keeps capability
  version `1`, resets development sessions, and restarts the stable JVM.

# Domain/PBT Tests

- PBT-18 runs at least 1,000 generated diagnostic maps and strings. Every
  accepted record is one line, within field/ring bounds, strips query/fragment
  data, redacts credential-like material, and contains only allowlisted keys;
  every other input is rejected without partial provider context.
- Provider process tests prove the diagnostic sentinel is absent from stdin
  and present only in the temporary Skill for a request that carries records.
- Coordinator/HTTP tests prove malformed records are rejected before queueing
  and accepted records remain transient.
- Browser tests prove the exact action failure, active-frame source check,
  ring deduplication/bounds, runtime clear, and parent-noise exclusion.

# Out of Scope

- Persistent analytics, replay, crash reporting, source maps, performance
  traces, screenshots, DOM snapshots, or recording every user interaction.
- Headers, cookies, request/response bodies, query values, connector secrets,
  structured product password/identifier fields, or provider reasoning.
- Capturing parent-window, DevTools, browser-extension, operating-system, or
  another tab's errors.
- Automatically spending a provider turn without an explicit user message.
- Public telemetry ingestion, third-party observability vendors, or a user
  console/network UI.
