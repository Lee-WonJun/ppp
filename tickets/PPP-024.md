# Outcome

The final three-minute video starts on Projects with workspace access already
established and shows one product evolving through real OAuth Codex turns:
Snake, product signup, visible validation repair, real signup/sign-in, a
SQLite-backed ranking, a multi-game platform, and Tetris. The viewer sees that
PPP changes browser behavior, server authority, and persistent data without a
terminal, fake provider, source-code tour, or developer-handoff cut.

# Scope

- Replace the Gallery demo narrative in the final capture plan with one
  continuous game-platform product story.
- Begin the visible recording on Projects. Shared-password entry remains a
  deployment prerequisite, not a filmed scene.
- Create Snake as a browser-owned timer and keyboard game.
- Add real generated-product signup/sign-in through the Kernel product-auth
  capability, then improve the form and surface useful validation and server
  errors after observing the first UI.
- Register and sign in a product user in the browser, including one deliberate
  invalid registration attempt whose error is visible and recoverable.
- Add an authenticated SQLite-backed Snake leaderboard whose data survives
  reload and is associated with the signed-in product user.
- Reframe the product as a game platform with Snake as one catalog entry while
  preserving accounts, scores, and the playable game.
- Add Tetris as a second timer/keyboard game without removing Snake or the
  server-owned product features.
- Provide a real-Codex-only browser rehearsal command for this exact sequence.
  Keep the deterministic fake provider as CI regression infrastructure, never
  as the final-video provider.
- Preserve the last provider thread after an exhausted repairable rejection so
  the next explicit correction continues from the exact failed context while
  rejected source remains inactive.
- Update the exact narration, subtitles, PRD, Devpost draft, README, rubric,
  release evidence, and traceability.
