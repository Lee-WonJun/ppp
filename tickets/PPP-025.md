# Outcome

A locally playable, sub-three-minute video shows the exact PPP-024 story using
real OAuth Codex output: Projects, Snake, product accounts, a visible account
error and recovery, authenticated SQLite ranking, platformization, and Tetris.
Long model waits are visibly marked as compressed; no fake-provider frames,
credentials, terminals, source, or private paths appear.

# Scope

- Add an explicit video-capture mode to the real `bb demo-live` path without
  changing its default verification behavior.
- Record the compiled browser at 1440x900 while the real Codex story runs.
- Record bounded scenario timeline metadata needed to cut generation waits.
- Produce a local final MP4 below 180 seconds and an English subtitle sidecar.
- Validate duration, dimensions, codec readability, the six scenario outcomes,
  and absence of fake-provider substitution.
- Update demo, release, Devpost, rubric, and traceability records with bounded
  local evidence.

# Acceptance Criteria

- [x] The recorded run uses `PPP_AI_PROVIDER=codex` with ChatGPT OAuth.
- [x] One project and one resumed provider thread pass all six scenarios.
- [x] The raw capture begins on Projects and records the compiled browser at
  1440x900.
- [x] The final MP4 is playable, at least 150 seconds, and shorter than 180
  seconds.
- [x] Generation waits are compressed or replaced by an explicit
  `Generation time compressed` card.
- [x] Snake timer/keyboard, account error and lifecycle, SQLite ranking reload,

- [x] No fake-provider output, login credential, terminal, source tree, diff,
  SQL, test output, session identifier, or local private path is visible.
- [x] The English subtitle file remains synchronized with the final runtime.
- [x] The artifact remains local until the owner separately authorizes upload.
