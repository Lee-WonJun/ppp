# Outcome

While real Codex is working, the web conversation replaces the single static
`Generating` detail with a calm sequence driven by actual JSONL lifecycle
events. Users can tell that the request is being understood, shaped, and
prepared without seeing hidden reasoning, generated source, paths, commands,
provider names, or a technical activity log.

# Scope

- Parse bounded Codex JSONL incrementally instead of waiting for process exit.
- Convert an allowlist of turn/item lifecycle metadata into fixed
  product-language progress summaries; never copy event text.
- Deliver deduplicated summaries through the existing `:turn/progress`
  WebSocket message and generated/fallback sidebar detail field.
- Retain the existing `Generating`, `Validating`, `Applying`, and `Applied`
  phase vocabulary, one-line layout, animated ellipsis, and reduced-motion
  behavior.
- Keep fake-provider, reply, clarify, repair, timeout, invalid JSONL, queue,
  and completed-turn behavior compatible.
- Update product, implementation, security, design, rubric, and traceability
  documents with the new bounded event-derived contract.

# Acceptance Criteria

- [x] A Codex `turn.started` event updates the requesting tab before the Codex
  process exits.
- [x] Supported reasoning, plan, and agent-message lifecycle events can advance
  the one-line detail while the public phase remains `Generating`.
- [x] Event text, reasoning content, source, command, path, token, model,
  diagnostic, and unknown event fields never reach the browser.
- [x] Duplicate or unsupported events do not create new visible progress.
- [x] JSONL byte limits, invalid-event failure behavior, thread continuity,
  timeout, and final structured-output validation remain intact.
- [x] The requesting tab displays the bounded `payload.detail`; absent or
  invalid detail falls back to the phase's existing copy.
- [x] Provider progress is volatile: it is not written to history, transcript
  summaries, session source, application logs, or checkpoints.
- [x] Focused JVM, CLJS, protocol, coordinator, and compiled-browser tests pass.
- [x] The stable owner-facing surface is promoted only after paired browser
  bundles pass, and breaking pre-release state is reset rather than versioned.

# Domain/PBT Tests

- Event minimization property: arbitrary JSONL item content maps either to one
  fixed allowlisted detail or to nothing; no input substring is copied.
- Streaming property: the first supported event is observable before process
  completion.
- Boundary property: progress details are one-line, allowlisted, bounded, and
  rejected or replaced when an untrusted server payload violates the contract.
- Compatibility tests retain invalid JSONL, oversized stream, timeout,
  thread-mismatch, queue, repair, and final-result behavior.
- Browser tests assert semantic status changes, not exact event count or timing.
