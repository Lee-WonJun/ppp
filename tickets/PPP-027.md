# Outcome

Hackathon judges can open `https://ppp.openai.slopbook.org`, enter the privately
delivered shared password, and use the real OAuth Codex product workspace over
HTTPS without sharing a runtime host, volume, domain, or application with the
existing Slopbook service.

# Scope

- Publish the verified `master` commit to the public PPP repository.
- Create one Coolify application on the Coolify `localhost` server from the
  repository Dockerfile. Keep the build and runtime on that host so deployment
  does not depend on a remote SSH worker.
- Configure exact-origin production access, the rolling 100-start provider
  budget, one replica, health/readiness endpoints, and resource limits.
- Mount separate persistent paths for session data and Codex credentials.
- Add the exact Porkbun DNS record for `ppp.openai.slopbook.org` without
  modifying `app.slopbook.org` or the Slopbook `worker3` application.
- Complete device OAuth in the persistent Codex volume.
- Verify HTTPS, password rejection/acceptance, Projects, readiness, and one
  real provider turn in a fresh browser.
- Re-render and verify the submission video with English narration, synchronized
  English subtitles, and an explicit explanation of Codex/GPT-5.6 use.
- Record sanitized deployment evidence without passwords, cookies, OAuth
  material, session data, prompts, provider JSONL, host keys, or raw logs.

# Acceptance Criteria

- [x] The deployed image is built from the recorded public Git commit.
- [x] Slopbook remains healthy on `worker3` and `https://app.slopbook.org`.
- [x] PPP runs as one application replica on Coolify `localhost` with port
  `8787` and bounded CPU/memory limits.
- [x] `/var/lib/ppp` and `/var/lib/codex` are distinct persistent mounts.
- [x] Production secrets exist only in Coolify secret storage or the Codex
  credential volume.
- [x] `ppp.openai.slopbook.org` resolves to the PPP worker and serves a valid
  HTTPS certificate.
- [x] Password URL fragments are disabled and an invalid password is rejected.
- [x] The shared password signs in and opens the common Projects workspace.
- [x] `/healthz` and `/readyz` succeed after persistent device OAuth.
- [x] One fresh real-Codex turn completes and visible progress advances without
  exposing provider-authored content.
- [x] The local submission video is under three minutes, uses English narration
  and English subtitles, and explains how Codex and GPT-5.6 are used.
- [x] Deployment, rollback, access delivery, and sanitized verification
  evidence are recorded.

# Domain/PBT Tests

- Reuse the `bb verify` release gate and PPP-022 access/budget properties.
- Public canary checks exact HTTPS origin, invalid and valid access outcomes,
  Projects rendering, clean console/network state, and real-provider readiness.
- Restart check proves the session and OAuth mounts survive container restart.

# Out of Scope

- Changing the existing Slopbook application, domain, data, or worker.
- Multi-replica deployment, public signup, judge identities, billing, or HA.
- Publishing the shared password, OAuth material, or a password-bearing URL.
- Devpost submission or public video upload.
