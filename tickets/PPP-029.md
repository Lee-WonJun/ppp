# Outcome

The README and submission inspiration explain PPP's technical lineage through
Emacs, Lisp, nREPL, and Deep Space 1 without overstating what happened in
space or implying that PPP exposes a public REPL.

# Scope

- Add a concise, linked lineage section to the README.
- Expand `docs/THESIS.md` with nREPL's tool/runtime contribution and its public
  endpoint security warning.
- Keep the Devpost inspiration aligned with the same accurate story.
- Explain why PPP is a product runtime rather than ordinary developer hot
  reload.
- Preserve the distinction between live diagnosis/recovery and a permanent
  flight-source patch.
- Update the public Devpost project copy without uploading or submitting.

# Acceptance Criteria

- [x] README connects Emacs, Lisp, nREPL, Deep Space 1, and PPP in plain

- [x] nREPL is credited as a tooling protocol for live runtimes while PPP is
