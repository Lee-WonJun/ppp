# Outcome

The submission film is rebuilt from a new recording made against the current
public judge server with its real Codex OAuth provider. No generated-product
frame from PPP-025 or any fake-provider fixture is reused.

# Scope

- Add a reproducible production-login capture path for the public judge URL.
- Create a new project during capture and drive the product through a coherent
  full-stack evolution using real Codex turns.
- Keep the public-server request, bounded progress, activation, and resulting
  product visually continuous; accelerate only genuine provider waits in edit.
- Rebuild the final English-narrated, English-captioned film entirely from the
  new live capture plus truthful explanatory graphics.
- Verify the new media, runtime outcomes, public readiness, and secret safety.

# Acceptance Criteria

- [x] Every generated-product frame comes from a new public-server recording.
- [x] The capture logs in without revealing or persisting the shared password.
- [x] A new project is created on the public server during the recorded run.
- [x] All product changes use the real configured Codex OAuth provider.
- [x] The story includes visible client behavior and a SQLite-backed server
      behavior or business-rule change.
- [x] No PPP-025 capture, fake-provider output, credential, session identifier,
      terminal, or private path appears in the final film.
- [x] Real waits remain visually continuous and may be transparently accelerated.
- [x] The final film is under 180 seconds with English narration and English
      burned plus embedded subtitles.
- [x] The public URL, health, readiness, final MP4 decode, subtitle timeline,
      audio stream, and secret scan pass.

# Domain/PBT Tests

- Assert the capture target origin is the public judge URL and rejects localhost.
- Assert the capture requires the production login contract and real provider.
- Assert the new recording manifest contains no PPP-025 input paths.
- Verify the requested project outcomes through semantic browser state and data.
- Probe and fully decode the rebuilt MP4; validate subtitle monotonicity and end.
- Scan capture logs, media metadata, subtitles, and build manifests for secrets.
